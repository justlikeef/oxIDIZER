Add this CA’s root certificate to your device’s trusted store. This is required once per device so that certificates issued by this CA are trusted automatically by your browser, OS, and command-line tools.

Operating Systems & Browsers
🪟 Windows
  1. Download ox-root-ca.crt using the button above.
  2. Double-click the file — the Certificate dialog opens.
  3. Click Install Certificate…
  4. Choose Local Machine and click Next.
  5. Select Place all certificates in the following store, then Browse and pick Trusted Root Certification Authorities.
  6. Click Next → Finish. Confirm the security prompt.
  7. Restart any open browsers.

Alternatively, import via PowerShell (run as Administrator):

Import-Certificate -FilePath ox-root-ca.crt `
    -CertStoreLocation Cert:\LocalMachine\Root

 macOS
  1. Download ox-root-ca.crt and double-click it.
  2. Keychain Access opens. Select the System keychain and click Add.
  3. Find the certificate in the list, double-click it to open it.
  4. Expand Trust and set When using this certificate to Always Trust.
  5. Close the dialog and enter your password when prompted.
  6. Restart browsers.

Or via Terminal (requires sudo):

sudo security add-trusted-cert \
    -d -r trustRoot \
    -k /Library/Keychains/System.keychain \
    ox-root-ca.crt

🐧 Linux — Debian / Ubuntu
  1. Download ox-root-ca.crt.
  2. Copy the certificate to the system trust store: 
    sudo cp ox-root-ca.crt \
    /usr/local/share/ca-certificates/ox-root-ca.crt
  3. Update the trust store: 
    sudo update-ca-certificates

Chrome and Chromium on Linux use the system NSS database. Run certutil as below if Chrome still shows warnings.

🐧 Linux — RHEL / Fedora / CentOS
  1. Download ox-root-ca.crt.
  2. The file must end in .crt (it does). Copy it: 
    sudo cp ox-root-ca.crt \
    /etc/pki/ca-trust/source/anchors/
  3. Rebuild the trust store: 
    sudo update-ca-trust extract
🛸 Firefox

Firefox maintains its own certificate store independent of the OS.

  1. Open Settings → Privacy & Security.
  2. Scroll to Certificates and click View Certificates…
  3. On the Authorities tab, click Import…
  4. Select the downloaded ox-root-ca.crt file.
  5. Check Trust this CA to identify websites and click OK.

Enterprise deployments can push this via policies.json or Group Policy.

🌐 Chrome & Edge

Chrome and Edge use the OS trust store on Windows and macOS. Follow the Windows or macOS instructions above.

On Linux, Chrome reads the NSS shared database. Add the cert with:

certutil -d sql:$HOME/.pki/nssdb \
    -A -t "CT,," \
    -n "ox Root CA" \
    -i ox-root-ca.crt

Create the database first if it doesn’t exist: mkdir -p ~/.pki/nssdb && certutil -d sql:$HOME/.pki/nssdb -N --empty-password

Command-Line Tools
● curl

Pass the CA certificate directly for a single request:

curl --cacert ox-root-ca.crt https://your-server/

Or add it permanently to curl’s default CA bundle. On Debian/Ubuntu this is handled by update-ca-certificates above. On macOS:

echo 'capath=/etc/ssl/certs/' >> ~/.curlrc

After running update-ca-certificates or the macOS security command, curl uses the system bundle automatically.

● wget

For a single request:

wget --ca-certificate=ox-root-ca.crt https://your-server/

Once installed at the OS level (see Linux instructions above), wget picks it up automatically.

● OpenSSL / s_client

Verify a TLS connection against this CA:

openssl s_client \
    -CAfile ox-root-ca.crt \
    -connect your-server:443

A successful handshake shows Verify return code: 0 (ok).
