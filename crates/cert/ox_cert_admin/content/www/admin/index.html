External systems can submit CSRs to the issue endpoint. When RA approval is required by policy, the request enters the Pending Queue above.

Submit a CSR for signing

POST /api/v1/certificates
Content-Type: application/json

{
  "csr": "<PEM-encoded CSR>",
  "profile": "server"
}

Or send the raw PEM body with Content-Type: application/pkcs10.

Returns 201 Created with the signed certificate, or 202 Accepted with a request_id when RA approval is required.

Renew a certificate

POST /api/v1/certificates/{id}/renew

Revoke a certificate

POST /api/v1/certificates/{id}/revoke
Content-Type: application/json

{
  "reason": "keyCompromise"
}

List all certificates

GET /api/v1/certificates

Poll for an approved certificate

GET /api/v1/ra/requests/{request_id}/certificate

Returns 200 with the certificate once approved and issued, or 202 while still pending.

Admin direct-sign (bypasses RA queue)

POST /api/v1/ra/sign
Content-Type: application/json

{
  "csr": "<PEM-encoded CSR>"
}

Also accepts a raw PEM body. Returns 201 Created with the signed certificate.

Queue management

GET  /api/v1/ra/pending                      # list pending requests
GET  /api/v1/ra/pending/{id}                 # get single request
POST /api/v1/ra/pending/{id}/approve         # approve (issues cert)
POST /api/v1/ra/pending/{id}/deny            # deny with reason
