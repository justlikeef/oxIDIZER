Certificate Authority

Internal PKI Services

CHECKING…

This is the internal Certificate Authority for your organisation. Use the functions below to enroll, manage, and verify certificates, or to set up trust on a new device.

Trust & Setup
🔒
Install Trusted Root Certificate
Add this CA’s root certificate to your device’s trusted store. Required once per device to avoid browser warnings on services signed by this CA.
/ca/trust/ — Windows · macOS · Linux · Firefox
📄
Download Root Certificate
Download the root CA certificate in PEM format for scripted deployments, server configuration, or manual trust store import.
/ca/root.crt — PEM format
Certificate Lifecycle
📝
Issue Certificate
Request a new TLS, client, or server certificate. Supports RSA and ECC key types, SANs, custom validity periods, and certificate profiles.
POST /api/v1/certificates
🔄
Renew Certificate
Renew an existing certificate before it expires. Preserves the subject and SANs while issuing a fresh validity period and optionally a new key.
POST /api/v1/certificates/{id}/renew
🚫
Revoke Certificate
Immediately revoke a compromised or decommissioned certificate. Updates the CRL and OCSP responder within seconds.
POST /api/v1/certificates/{id}/revoke
📋
Certificate Inventory
Browse all issued certificates, filter by status, expiry, or subject, and view the full audit trail for each certificate.
GET /api/v1/certificates
Revocation & Status
✅
Certificate Status (OCSP)
Check in real time whether a certificate has been revoked using the Online Certificate Status Protocol. Used automatically by browsers and TLS clients.
/ocsp/ — RFC 6960
🗎
Certificate Revocation List
Download the current CRL listing all revoked certificates. Published on a regular schedule and available for offline or policy-based revocation checking.
/crl — RFC 5280
Automated Enrollment
🤖
ACME Protocol
Automate certificate issuance and renewal using the ACME protocol (RFC 8555). Compatible with certbot, acme.sh, Caddy, and other ACME clients. Supports http-01 and dns-01 challenges.
/acme/directory — RFC 8555
🌐
EST — Enrollment over Secure Transport
Enroll for certificates using EST (RFC 7030). Supports simple enroll, re-enroll, and CA certificate retrieval for compliant devices and infrastructure.
/.well-known/est/ — RFC 7030
🔌
SCEP Challenge Passwords
Provision one-time challenge passwords for SCEP-capable devices. Generate a challenge here, then supply the password to the device during its SCEP enrollment.
/api/v1/scep/challenges
🔢
SSH Certificates
Issue OpenSSH certificates for user and host identity. Enables short-lived SSH access without distributing individual public keys across servers.
/api/v1/ssh/ — OpenSSH format
Export & Integration
📦
PKCS#12 Export
Export a certificate and its private key as a password-protected PKCS#12 (.p12 / .pfx) bundle for use in browsers, Java keystores, or Windows certificate stores.
POST /api/v1/certificates/{id}/p12
📱
Registration Authority
Submit certificate requests that require manual approval before issuance. Administrators review and approve or deny pending requests via the admin interface.
/api/v1/certificates — RA workflow
Administration
⚙️
CA Administration
Manage the CA hierarchy, view issued certificates, inspect the audit log, configure profiles and policies, and manage the CA key material.
/ca/admin/ — full management interface
👥
Certificate Status
Inspect the server’s root CA certificate and verify the trust status of the client certificate presented with this request.
/ca/certinfo/
👥
Trust Management
Configure cross-certification, trust anchors, and external CA relationships. Manage which CAs are trusted for policy enforcement.
/api/v1/trust
🔎
Audit Log
Review the complete audit trail of all CA operations: issuance, renewal, revocation, key access, and administrator actions.
/api/v1/audit
📈
System Status
View the health and operational status of the CA service, including key store accessibility, database connectivity, and CRL freshness.
/status
