# ox_cert_ssh

**Purpose:** SSH Certificate Authority — issues OpenSSH user and host certificates.
These are not X.509 certificates; they use OpenSSH's native binary certificate format.

---

## Phase
`Content`

## Routes

| Method | Path | Description |
|---|---|---|
| `POST` | `/api/v1/ssh/sign` | Sign an SSH public key, return an SSH certificate |
| `POST` | `/api/v1/ssh/renew` | Renew an existing SSH certificate |
| `GET` | `/api/v1/ssh/ca/user` | Return user CA public key (for `TrustedUserCAKeys`) |
| `GET` | `/api/v1/ssh/ca/host` | Return host CA public key (for `known_hosts`) |
| `GET` | `/api/v1/ssh/config` | Return recommended `sshd_config` and `ssh_config` snippets |

Route registration: `"GET,POST /api/v1/ssh/*"`.

## Crate Dependencies

| Crate | Purpose |
|---|---|
| `ox_cert_core` | `KeyStore`, `CertStore`, `SshCertBuilder`, `SshCertRecord`, `CertError` |
| `ssh-key` | Parse OpenSSH public key format, build OpenSSH certificate binary |
| `uuid` (v4) | Not used for SSH serial (u64); used for audit event IDs |
| `serde` / `serde_json` | Request/response serialization |
| `time` | Validity period computation and duration parsing |

## Config

```rust
#[derive(Debug, Deserialize)]
pub struct SshConfig {
    pub tenant_id: String,
    pub store: CertStoreConfig,
    pub keystore: KeyStoreConfig,
    pub user_ca: SshCaConfig,
    pub host_ca: SshCaConfig,
    pub user: SshPrincipalPolicy,
    pub host: SshPrincipalPolicy,
}

#[derive(Debug, Deserialize)]
pub struct SshCaConfig {
    /// Key ID in KeyStore. Key is generated on first init if missing.
    pub key_id: String,
    pub key_type: SshCaKeyType,   // Ed25519 | EcdsaP256 | EcdsaP384
    /// Default validity duration string (e.g., "16h", "30d").
    pub default_validity: String,
}

#[derive(Debug, Deserialize)]
pub enum SshCaKeyType { Ed25519, EcdsaP256, EcdsaP384 }

#[derive(Debug, Deserialize)]
pub struct SshPrincipalPolicy {
    /// Glob patterns for allowed principals (e.g., ["*"] or ["*.example.com"]).
    pub allowed_principals: Vec<String>,
    /// Default OpenSSH extensions for user certs (empty map for host certs).
    pub default_extensions: std::collections::HashMap<String, String>,
    /// Default critical options (usually empty).
    pub default_critical_options: std::collections::HashMap<String, String>,
    /// Maximum validity duration string (e.g., "24h" for users, "8760h" for hosts).
    pub max_validity: Option<String>,
}
```

## SSH Serial Counter

SSH certificate serials are u64 (per the OpenSSH wire format), not UUID. They are
generated by atomically incrementing `ssh_serial_counter.next_serial` for the tenant:

```sql
UPDATE ssh_serial_counter
SET next_serial = next_serial + 1
WHERE tenant_id = $1
RETURNING next_serial;
```

If no row exists for the tenant, insert `(tenant_id, 1)` first. The returned value is
the serial for this certificate.

## Input TaskState Fields (POST /api/v1/ssh/sign)

| Field | Description |
|---|---|
| `request.body` | JSON: `{ public_key, cert_type, principals[], validity?, key_id?, critical_options?, extensions? }` |

### Request Body Schema

```json
{
  "public_key":       "ssh-ed25519 AAAA...",
  "cert_type":        "user",
  "principals":       ["alice", "admin"],
  "validity":         "16h",
  "key_id":           "alice@example.com",
  "critical_options": {},
  "extensions":       { "permit-pty": "", "permit-port-forwarding": "" }
}
```

`public_key` and `cert_type` are required. `principals` is required and must be non-empty.
`validity` defaults to the profile's `default_validity`. If `extensions` is omitted,
`default_extensions` from config is used.

## Processing (POST /api/v1/ssh/sign)

1. Parse request body.
2. Select CA config: `user_ca` if `cert_type = "user"`, `host_ca` if `"host"`.
3. Parse `public_key` string using `ssh-key` crate.
4. Validate principals against `allowed_principals` glob list. Return 403 if any
   principal is not matched by any allowed pattern.
5. Resolve validity: parse `validity` string (e.g., `"16h"` → `Duration`) using
   `humantime` or equivalent. Cap at `max_validity` if configured.
6. Compute `valid_after = now`, `valid_before = now + duration`.
7. Obtain next SSH serial via `store.get_next_ssh_serial(tenant_id)`.
8. Resolve effective extensions: use request `extensions` if provided, else
   `config.user.default_extensions` (or empty for host certs).
9. Build `SshCertBuilder`:
   - `cert_type`, `key_id`, principals, validity window, serial.
   - Add all extensions and critical options.
10. Call `builder.sign(subject_pubkey_bytes, key_store, tenant_id, ca_key_id)`.
    Returns `SshCertRecord`.
11. `store.store_ssh_cert(tenant_id, &record)`.
12. `store.store_audit_event(tenant_id, AuditEvent { action: SshSign, ... })`.
13. Set `response.status = "201"`, `response.body` to JSON.

---

## Processing (GET /api/v1/ssh/ca/user and /host)

1. Look up the CA key public key: `key_store.public_key(tenant_id, ca_key_id)`.
2. Format as OpenSSH authorized_keys line: `{key_type} {base64} {comment}`.
3. Return as `text/plain`.

## Processing (GET /api/v1/ssh/config)

Return a JSON object with recommended SSH configuration snippets:

```json
{
  "sshd_config_snippet": "TrustedUserCAKeys /etc/ssh/trusted_user_cas",
  "known_hosts_snippet": "@cert-authority *.example.com ssh-ed25519 AAAA...",
  "user_ca_public_key": "ssh-ed25519 AAAA...",
  "host_ca_public_key": "ssh-ed25519 AAAA...",
  "notes": {
    "TrustedUserCAKeys": "Add user_ca_public_key contents to this file on each SSH server",
    "HostCertificate": "Host certs must be signed separately using POST /api/v1/ssh/sign with cert_type=host and the server's host public key"
  }
}
```

## Processing (POST /api/v1/ssh/renew)

| Field | Description |
|---|---|
| `request.body` | JSON: `{ serial, validity? }` |

1. `store.get_ssh_cert_by_serial(tenant_id, serial)` → 404 if not found.
2. Check `valid_before > now - 5min` (reject renewal of long-expired certs).
3. Reuse principals, extensions, and critical options from the original record.
4. Sign a new cert with the same public key (from `record.public_key`), new validity.
5. Store new record. Audit.

## Error Cases

| Condition | HTTP | Code |
|---|---|---|
| Missing `public_key` or `cert_type` | 400 | `INVALID_REQUEST` |
| Unparseable public key | 400 | `INVALID_REQUEST` |
| Principal not in allowed list | 403 | `POLICY_VIOLATION` |
| `validity` exceeds `max_validity` | 400 | `POLICY_VIOLATION` |
| CA key not initialized | 503 | `CA_NOT_READY` |
| Storage failure | 500 | `INTERNAL_ERROR` |
