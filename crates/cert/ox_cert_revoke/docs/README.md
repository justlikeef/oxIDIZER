# ox_cert_revoke

Revokes a certificate by serial number, recording the revocation reason and timestamp.

---

## Phase

`Content`

## Routes

| Method | Path | Description |
|---|---|---|
| `POST` | `/api/v1/certificates/{serial}/revoke` | Revoke certificate by serial |

---

## Config Reference

```rust
pub struct CertRevokeConfig {
    pub tenant_id: String,
    pub store: CertStoreConfig,
}
```

This is the simplest plugin config — only tenant and store are needed. No KeyStore
access is required for revocation.

---

## Request Body

```json
{
  "reason": "key_compromise"
}
```

`reason` is optional and defaults to `unspecified`. Valid values:

| Value | RFC 5280 Code |
|---|---|
| `unspecified` | 0 |
| `key_compromise` | 1 |
| `ca_compromise` | 2 |
| `affiliation_changed` | 3 |
| `superseded` | 4 |
| `cessation_of_operation` | 5 |
| `certificate_hold` | 6 |
| `privilege_withdrawn` | 9 |

---

## Output

```json
{ "data": { "serial": "uuid", "revoked_at": "2026-04-22T10:00:00Z", "reason": "key_compromise" }, "meta": { ... } }
```

HTTP 200 on success.

---

## Error Cases

| Condition | HTTP | Code |
|---|---|---|
| Serial not found | 404 | `NOT_FOUND` |
| Certificate already revoked | 409 | `ALREADY_REVOKED` |
| Unknown reason string | 400 | `INVALID_REQUEST` |
| Storage failure | 500 | `INTERNAL_ERROR` |

---

## Implementation Notes

- Revocation marks the database record `status = Revoked` and records `revoked_at` and
  `revocation_reason`. It does not generate a new CRL — that is `ox_cert_crl`'s job.
- An `AuditEvent` with `action = Revoke` is stored on every successful revocation.
- The next CRL generated by `ox_cert_crl` will automatically include this serial.
